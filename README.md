# DANO 2025 — Воронка авиабилетов (Т-Банк Авиа)

Проект выполнен в рамках Т-Интенсива по продуктовой аналитике: выбрать датасет, провести разведочный анализ (EDA), оформить ключевые выводы в презентацию (аналитическая записка на 3–4 слайда), сформулировать 2–3 продуктовые гипотезы и описать дизайн эксперимента для проверки одной выбранной гипотезы (саму проверку на данных проводить не требовалось).

**Датасет:** №8 «Т-Банк: воронка авиабилетов» (период: январь–сентябрь 2025)  
**Источник датасетов:** https://dano.hse.ru/data2025

---

## Контекст и цель
Сервис «Авиа» внутри экосистемы Т-Банка: события по пользователям/сессиям на этапах воронки (поиск -> выдача -> оффер -> чекаут -> покупка).

**Цель проекта:**
1) Найти ключевые потери воронки и оценить конверсии по шагам  
2) Понять поведенческие паттерны на проблемных этапах
3) Предложить продуктовые улучшения и спланировать эксперимент для проверки выбранной гипотезы

---

## Ключевые результаты
- Итоговая конверсия в покупку: **~6%** (**2 364 / 37 756** сессий).
- Наибольший провал на шаге: **«Выдача -> Оффер»** — наибольшая потеря пользователей **~52%**.
- Анализ времени на странице выдачи показал разные поведенческие сценарии:
  - **10–30 сек** — максимальная конверсия в оффер (**~0.85**),
  - **≤10 сек** — быстрый уход (**~0.2**),
  - при **очень долгих** сессиях конверсия снижается.
- По месяцам объём сессий стабилен (≈ **4000–4800/мес**), при этом заметен провал конверсии в покупку в **июле** и восстановление в **августе**.

---

## Что сделано (EDA)
- Построена воронка по шагам и посчитаны конверсии между этапами
- Проанализировано время до перехода в оффер и сегментация по времени на выдаче
- Посмотрена динамика по месяцам (объём сессий и конверсия)
- Основные выводы и графики вынесены в презентацию

---

## Продуктовые гипотезы (2–3)
1) **Персонализированная выдача / рекомендательный блок**  
   Предположение: повышение релевантности первых предложений и блок «Рекомендуем» увеличит переходы в оффер и снизит долю быстрых уходов с выдачи.

2) **Персонализированный ретеншн (пуш/сообщения) для пользователей, не дошедших до покупки**  
   Предположение: коммуникации по “почти созревшим” пользователям (которые дошли до выдачи/оффера, но не купили) увеличат возвраты и итоговую конверсию в покупку.

3) **Сезонные улучшения/предложения для периода падения конверсии**  
   Предположение: точечные сезонные условия/коммуникации в период ухудшения показателей (например, июль) помогут компенсировать падение конверсии.

---

## Выбранная гипотеза
**Гипотеза №1: персонализированная выдача / рекомендательный блок.**  
Причина выбора: она нацелена на самый проблемный шаг воронки **«Выдача -> Оффер»**, где наблюдается максимальная потеря пользователей (~52%).

---

## Дизайн A/B-теста

**Группы:**
- **A (Control):** текущая выдача
- **B (Test):** персонализированная выдача + блок «Рекомендуем» (учёт истории поиска/кликов)

**Популяция:** все пользователи/сессии, дошедшие до шага **«Выдача»**

**Primary metric:**
- `CR(Выдача -> Оффер)` (по пользователям и/или по сессиям)

**Secondary metrics:**
- доля “коротких” визитов на выдаче: `share(time_on_search ≤ 10s)`
- downstream-метрики, чтобы убедиться, что рост кликов не ухудшает покупки:
  - `CR(Оффер -> Чекаут)`
  - `CR(Чекаут -> Покупка)`
